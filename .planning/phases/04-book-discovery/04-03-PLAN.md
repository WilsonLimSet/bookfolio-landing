1. CREATE Services/BookActionService.swift — static methods for book actions:

```swift
struct BookMetadata {
    let openLibraryKey: String
    let title: String
    let author: String?
    let coverUrl: String?
}
```

Methods:

a. `static func addToWantToRead(userId: UUID, book: BookMetadata) async throws`:
   - Insert into want_to_read: user_id, open_library_key, title, author, cover_url
   - Fire-and-forget activity log: NewActivity with action_type "want_to_read" and book fields
   - Use a private Encodable struct for the insert (no server-generated id)

b. `static func removeFromWantToRead(userId: UUID, bookKey: String) async throws`:
   - Delete from want_to_read WHERE user_id AND open_library_key

c. `static func addToCurrentlyReading(userId: UUID, book: BookMetadata) async throws`:
   - Insert into currently_reading: user_id, open_library_key, title, author, cover_url
   - Fire-and-forget: remove from want_to_read (auto-cleanup)
   - Fire-and-forget activity log: NewActivity with action_type "started_reading"

d. `static func removeFromCurrentlyReading(userId: UUID, bookKey: String) async throws`:
   - Delete from currently_reading WHERE user_id AND open_library_key

e. `static func checkBookStatus(userId: UUID, bookKey: String) async -> (isWantToRead: Bool, isCurrentlyReading: Bool, userBook: UserBook?)`:
   - Query want_to_read, currently_reading, user_books in parallel (all with try?)
   - Return tuple of booleans and optional UserBook

IMPORTANT: Activity logging should use `Task { try? await ... }` fire-and-forget pattern — don't block the main action on activity insert.
IMPORTANT: The want_to_read auto-cleanup when starting reading should also be fire-and-forget.

2. CREATE WantToReadButton.swift:

Properties:
- `let book: BookMetadata`
- `@Binding var isWantToRead: Bool`
- `@EnvironmentObject var authService: AuthService`
- `@State private var isLoading = false`

Display:
- Not active: Button "Want to Read" with bookmark icon, outlined style
- Active: Button "Want to Read" with bookmark.fill icon, amber filled style
- Loading: ProgressView replacing icon

Action:
- Get userId from authService
- Optimistic: toggle isWantToRead immediately
- If adding: call BookActionService.addToWantToRead
- If removing: call BookActionService.removeFromWantToRead
- On error: revert isWantToRead, show error (print for now)

3. CREATE CurrentlyReadingButton.swift:

Properties:
- `let book: BookMetadata`
- `@Binding var isCurrentlyReading: Bool`
- `@Binding var isWantToRead: Bool` (to update when starting reading auto-removes want-to-read)
- `@EnvironmentObject var authService: AuthService`
- `@State private var isLoading = false`

Display:
- Not active: Button "Start Reading" with book icon, outlined style
- Active: Button "Reading" with book.fill icon, blue filled style

Action:
- Get userId from authService
- Optimistic: toggle isCurrentlyReading immediately. If starting, also set isWantToRead = false
- If adding: call BookActionService.addToCurrentlyReading
- If removing: call BookActionService.removeFromCurrentlyReading
- On error: revert both states

IMPORTANT: Both buttons use @Binding so the parent (BookDetailView) owns the state and can coordinate between them.
