Web app openLibrary.ts functions to replicate:

1. **searchBooks(query)** → GET `https://openlibrary.org/search.json?q={query}&limit=20&fields=key,title,author_name,cover_i,cover_edition_key,first_publish_year,edition_count`
   - Filter to books with cover_i
   - Sort: title match priority (all query words in title), then by edition_count
   - Return top 8 results
   - Handle translated titles: if title has non-Latin chars and doesn't contain query words, use capitalized query as display title

2. **getBookDetails(workKey)** → 3 parallel fetches:
   - `https://openlibrary.org{workKey}.json` → OLWork
   - `https://openlibrary.org{authorKey}.json` → OLAuthor (from work.authors[0].author.key)
   - `https://openlibrary.org{workKey}/editions.json?limit=50` → OLEditionsResponse
   - Cover priority: work.covers[0] → bestEdition.covers[0]
   - Description fallback: work.description → bestEdition.description → work.excerpts → Google Books API
   - Return BookDetails struct

3. **getEditions(workKey)** → GET `https://openlibrary.org{workKey}/editions.json?limit=200`
   - Score by popularity: English +15, has ISBN +5, has cover +3, has publisher +1, non-English -10
   - Deduplicate by cover ID
   - Return top 24 editions with unique covers

4. **fetchWorkSubjects(workKey)** → GET `https://openlibrary.org{workKey}.json`
   - Return work.subjects array

5. **detectCategory(subjects)** → local scoring algorithm:
   - NYT tags (weight 3): "nyt:fiction" → fiction, "nyt:nonfiction" → nonfiction
   - BISAC (weight 2): "fiction /" → fiction, "biography & autobiography" etc → nonfiction
   - Strong markers (weight 2): "true crime", "biography", "memoir" → nonfiction; exact "fiction" → fiction
   - Genre keywords (weight 1): "fantasy", "thriller", "mystery" → fiction; "history", "science" → nonfiction
   - Tie-break: nonfiction wins (OpenLibrary over-tags fiction)

6. **getGoogleBooksDescription(title, author)** → GET `https://www.googleapis.com/books/v1/volumes?q={query}&maxResults=1`
   - Fallback when OpenLibrary description is missing or non-English

Define result types:

```swift
struct BookSearchItem: Identifiable {
    let id: String  // work key
    let key: String
    let title: String
    let author: String?
    let coverUrl: String?

    let editions: Int

struct BookDetails {
    let key: String
    let title: String
    let author: String?
    let translator: String?
    let coverUrl: String?
    let description: String?
    let firstPublishYear: Int?
    let subjects: [String]
    let pageCount: Int?
}

struct BookEditionItem: Identifiable {
    let id: String  // edition key
    let key: String
    let title: String?
    let coverUrl: String?
    let publisher: String?
    let year: String?
}
```

Static methods:

1. `static func searchBooks(query: String) async throws -> [BookSearchItem]`:
   - URL: `https://openlibrary.org/search.json?q=\(encoded)&limit=20&fields=key,title,author_name,cover_i,cover_edition_key,first_publish_year,edition_count`
   - Decode as OLSearchResponse
   - Filter: only results with coverId != nil
   - Sort: results where ALL query words appear in title (case-insensitive) first, then by edition_count desc
   - Take first 8
   - Map to BookSearchItem with coverUrl from coverId.openLibraryCoverURL(size: .medium)
   - Handle translated titles: if title contains non-Latin characters (regex `[^\u{0000}-\u{024F}\\s]`), use capitalized query as display title

2. `static func getBookDetails(workKey: String) async throws -> BookDetails?`:
   - Fetch work, author, and editions in parallel using async let:
     a. Work: GET `https://openlibrary.org\(workKey).json` decoded as OLWork
     b. Author: GET `https://openlibrary.org\(authorKey).json` decoded as OLAuthor (authorKey from work.authors?[0].author.key). Wrap in try? — author fetch is optional.
     c. Editions: GET `https://openlibrary.org\(workKey)/editions.json?limit=50` decoded as OLEditionsResponse
   - Find best edition: score editions by popularity (see getEditions scoring), pick highest
   - Cover: work.covers?.first?.openLibraryCoverURL(size: .large) ?? bestEdition.covers?.first?.openLibraryCoverURL(size: .large)
   - Description (4-tier fallback):
     a. work.description?.text
     b. bestEdition description (need to add OLDescription? to OLEdition if not present — check first)
     c. work excerpts (not in current Swift types — skip if not available)
     d. Google Books fallback via getGoogleBooksDescription
   - Subjects: work.subjects filtered to exclude NYT prefixes ("nyt:"), take first 6
   - Build and return BookDetails

3. `static func getEditions(workKey: String) async throws -> [BookEditionItem]`:
   - URL: `https://openlibrary.org\(workKey)/editions.json?limit=200`
   - Decode as OLEditionsResponse
   - Score each edition:
     - Has covers: proceed (skip editions without covers)
     - English language (check if edition has languages containing "/languages/eng"): +15
     - Non-English language data: -10
     - No language data: +5 (or -5 if title has non-Latin chars)
     - Has ISBN (isbn13 or isbn10 not nil/empty): +5
     - Has publisher (publishers not nil/empty): +1
   - Sort by score descending
   - Deduplicate by first cover ID (track seen cover IDs in Set)
   - Take first 24
   - Map to BookEditionItem with coverUrl from covers.first.openLibraryCoverURL(size: .medium)

   IMPORTANT: OLEdition doesn't have a `languages` field in the Swift model. You need to either:
   a. Add `languages: [OLLanguageRef]?` to OLEdition (with `struct OLLanguageRef: Codable { let key: String }`)
   b. OR skip language-based scoring and use a simpler heuristic

   Option (a) is better — add the languages field to OLEdition in OpenLibraryTypes.swift with a CodingKey mapping. This is a small addition to an existing file.

4. `static func fetchWorkSubjects(workKey: String) async throws -> [String]`:
   - URL: `https://openlibrary.org\(workKey).json`
   - Decode as OLWork
   - Return work.subjects ?? []

5. `static func detectCategory(subjects: [String]) -> BookCategory?`:
   - Port the weighted scoring algorithm from web:
     - var fictionScore = 0, nonfictionScore = 0
     - For each subject (lowercased):
       - NYT tags (weight 3): starts with "nyt:" → check "fiction" or "nonfiction"
       - BISAC (weight 2): starts with "fiction /" or "fiction/" → +2 fiction. Contains "biography & autobiography", "business", "self-help", starts with "history /", "science /", "psychology /" → +2 nonfiction
       - Strong markers (weight 2): exact "true crime", "biography", "memoir" etc → +2 nonfiction. Exact "fiction" → +2 fiction
       - Genre keywords (weight 1): "fantasy", "thriller", "mystery", "romance", "horror" etc → +1 fiction. "history", "science", "psychology" etc → +1 nonfiction
     - If both 0 → return nil
     - If tied → return .nonfiction
     - Return higher score category

6. `static func getGoogleBooksDescription(title: String, author: String?) async -> String?`:
   - Query: "\(title) \(author ?? "")"
   - URL: `https://www.googleapis.com/books/v1/volumes?q=\(encoded)&maxResults=1`
   - Define a simple Decodable struct for Google Books response:
     ```swift
     struct GoogleBooksResponse: Codable {
         let items: [GoogleBooksItem]?
     }
     struct GoogleBooksItem: Codable {
         let volumeInfo: GoogleVolumeInfo?
     }
     struct GoogleVolumeInfo: Codable {
         let description: String?
     }
     ```
   - Return items?.first?.volumeInfo?.description
   - Wrap in try? — failure returns nil

HTTP helper: Use URLSession.shared.data(from: url). Define a private helper:
```swift
private static func fetch<T: Decodable>(_ url: URL) async throws -> T {
    let (data, _) = try await URLSession.shared.data(from: url)
    return try JSONDecoder().decode(T.self, from: data)
}
```

IMPORTANT: All URL construction should use URLComponents for proper encoding, or at minimum percent-encode the query parameter.
IMPORTANT: OLEdition in OpenLibraryTypes.swift needs a `languages` field added. Add `struct OLLanguageRef: Codable { let key: String }` and `languages: [OLLanguageRef]?` to OLEdition with CodingKey.
IMPORTANT: The `description` field on OLEdition also needs adding if not present — add as `OLDescription?` with CodingKey mapping (OpenLibrary editions can have descriptions too).
IMPORTANT: Use `try?` liberally for optional fetches (author, Google Books) — these should not fail the entire operation.
