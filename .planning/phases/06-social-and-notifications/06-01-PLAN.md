Web app comment behavior (CommentSection.tsx):
- Load comments for review: select from review_comments WHERE review_id, ordered by created_at ASC, limit 50
- Post comment: insert into review_comments (user_id, review_id, comment_text)
- Delete own comment: delete from review_comments WHERE id
- Notification: insert into notifications (user_id=review owner, type="comment", from_user_id, book_title, review_id) — only if commenter != review owner
- Max 500 chars for comment text
- Shows "Sign in to comment" for unauthenticated users

- FeedService.toggleLike creates notifications for likes
- ProfileService uses static methods pattern

1. CREATE Services/CommentService.swift — enum with static methods:

a. `static func loadComments(reviewId: UUID) async throws -> [(comment: ReviewComment, profile: Profile)]`:
   - Query: `supabase.from("review_comments").select("*, profiles(id, username, avatar_url)").eq("review_id", reviewId).order("created_at").limit(50)`
   - Use a private Decodable row type for the join response:
     ```swift
     private struct CommentRow: Decodable {
         let id: UUID
         let userId: UUID
         let reviewId: UUID
         let commentText: String
         let createdAt: Date
         let profiles: ProfileRow

         struct ProfileRow: Decodable {
             let id: UUID
             let username: String?
             let avatarUrl: String?
         }
     }
     ```
   - Map to tuples of (ReviewComment, Profile)

b. `static func postComment(userId: UUID, reviewId: UUID, text: String, reviewOwnerId: UUID, bookTitle: String?) async throws`:
   - Insert into review_comments: user_id, review_id, comment_text
   - If userId != reviewOwnerId, fire-and-forget insert notification:
     type="comment", user_id=reviewOwnerId, from_user_id=userId, book_title=bookTitle, review_id=reviewId

c. `static func deleteComment(commentId: UUID) async throws`:
   - Delete from review_comments WHERE id = commentId

IMPORTANT: Use private Encodable structs for inserts (no server-generated fields).
IMPORTANT: Notification insert is fire-and-forget: `Task { @Sendable in try? await ... }`
