  <action>
1. Create Config.swift:
   - Read the existing .env.local from the web app to get the Supabase URL and anon key
   - Store as static constants (these are public anon keys, safe to embed — Supabase uses RLS for security)
   - Include app URL scheme for OAuth redirects
   ```swift
   enum Config {
       static let supabaseURL = "https://[project-ref].supabase.co"
       static let supabaseAnonKey = "[anon-key]"
       static let appScheme = "bookfolio"
       static let redirectURL = URL(string: "bookfolio://auth-callback")!
   }
   ```

2. Create SupabaseService.swift:
   - Global `let supabase` singleton (simple, following supabase-swift docs pattern)
   - Configure with PKCE auth flow for mobile security
   - Set redirectToURL for OAuth callbacks
   ```swift
   import Supabase

   let supabase = SupabaseClient(
       supabaseURL: URL(string: Config.supabaseURL)!,
       supabaseKey: Config.supabaseAnonKey,
       options: SupabaseClientOptions(
           auth: .init(
               flowType: .pkce,
               redirectToURL: Config.redirectURL
           )
       )
   )
   ```

3. Create BookfolioApp.swift:
   - Minimal @main App struct with WindowGroup
   - ContentView placeholder showing "Bookfolio" text

   ```swift
   @main
   struct BookfolioApp: App {
       var body: some Scene {
           WindowGroup {
               Text("Bookfolio")
           }
       }
   }
   ```

IMPORTANT: Read the actual values from the web app's .env.local file for the Supabase URL and anon key. These are the SAME backend — the iOS app connects to the existing Supabase project.
