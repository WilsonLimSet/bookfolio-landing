Navigation architecture (from research):
- TabView with 5 tabs: Feed, Search, Profile, Leaderboard, Lists
- Each tab wraps content in its own NavigationStack with independent NavigationPath
- AppRouter (ObservableObject) manages selected tab + per-tab TabRouter instances
- TabRouter (ObservableObject) owns NavigationPath for programmatic push/pop
- AppRoute (Hashable enum) defines all navigable destinations
- RouteDestination ViewModifier maps AppRoute cases to destination views
- iOS 16 patterns: @StateObject for ownership, @EnvironmentObject for injection, ObservableObject (not @Observable)

Tab configuration:
- Feed: book.fill icon
- Search: magnifyingglass icon
- Profile: person.fill icon
- Leaderboard: trophy.fill icon
- Lists: list.bullet icon

2. Create Navigation/TabRouter.swift — Per-tab navigation state:
```swift
class TabRouter: ObservableObject {
    @Published var path = NavigationPath()

    func push(_ route: AppRoute) { path.append(route) }
    func pop() { guard !path.isEmpty else { return }; path.removeLast() }
    func popToRoot() { path = NavigationPath() }
}
```

3. Create Navigation/AppRouter.swift — App-wide navigation coordinator:
```swift
enum Tab: Int, CaseIterable {
    case feed, search, profile, leaderboard, lists

    var title: String { ... }
    var icon: String { ... }
}

class AppRouter: ObservableObject {
    @Published var selectedTab: Tab = .feed
    let feedRouter = TabRouter()
    let searchRouter = TabRouter()
    let profileRouter = TabRouter()
    let leaderboardRouter = TabRouter()
    let listsRouter = TabRouter()

    func router(for tab: Tab) -> TabRouter { ... }
    func navigate(to tab: Tab, route: AppRoute? = nil) { ... }
    func popToRoot(tab: Tab) { ... }
}
```

4. Create Navigation/RouteDestination.swift — ViewModifier mapping routes to views:
   - For now, all destinations show placeholder views with the route info displayed
   - Example: `.bookDetail(bookKey)` shows Text("Book Detail: \(bookKey)")

IMPORTANT: Use @Published for AppRouter properties. TabRouter instances should NOT be @Published since they are reference types and their internal @Published path already triggers updates. Instead, store them as plain `let` properties on AppRouter.

2. Update BookfolioApp.swift to use the navigation shell:
```swift
@main
struct BookfolioApp: App {
    @StateObject private var appRouter = AppRouter()

    var body: some Scene {
        WindowGroup {
            TabView(selection: $appRouter.selectedTab) {
                FeedTab()
                    .tabItem { Label(Tab.feed.title, systemImage: Tab.feed.icon) }
                    .tag(Tab.feed)
                SearchTab()
                    .tabItem { Label(Tab.search.title, systemImage: Tab.search.icon) }
                    .tag(Tab.search)
                ProfileTab()
                    .tabItem { Label(Tab.profile.title, systemImage: Tab.profile.icon) }
                    .tag(Tab.profile)
                LeaderboardTab()
                    .tabItem { Label(Tab.leaderboard.title, systemImage: Tab.leaderboard.icon) }
                    .tag(Tab.leaderboard)
                ListsTab()
                    .tabItem { Label(Tab.lists.title, systemImage: Tab.lists.icon) }
                    .tag(Tab.lists)
            }
            .environmentObject(appRouter)
        }
    }
}
```

IMPORTANT: Each tab must have its OWN NavigationStack — do NOT wrap the TabView in a single NavigationStack (that breaks tab navigation independence). The `.environmentObject(appRouter)` must be on the TabView or above, NOT on individual tabs.
