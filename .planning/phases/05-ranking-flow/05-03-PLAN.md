Web app binary search algorithm:
- State: low=0, high=tierBooks.length, compareIndex=floor((low+high)/2)
- User prefers new book → high = compareIndex (new book is better)
- User prefers existing book → low = compareIndex + 1 (existing is better)
- When low >= high → done. finalPosition = higherTierBooks.count + low + 1
- Skip → place at mid position: higherTierBooks.count + floor((low+high)/2) + 1
- tierBooks = userBooksCache[category].filter(tier == selectedTier)
- higherTierBooks = userBooksCache[category].filter(tierOrder[tier] < tierOrder[selectedTier])
- 400ms animation delay between comparisons in web app

Core value: "The ranking flow must feel incredible on iOS — haptic feedback on every comparison, buttery spring animations, gesture-driven interactions."

Binary search logic — `func handlePrefer(preferNew: Bool)`:
```swift
func handlePrefer(preferNew: Bool) {
    guard !isAnimating else { return }
    isAnimating = true

    // Haptic
    UIImpactFeedbackGenerator(style: .medium).impactOccurred()

    // Show selection
    selectedSide = preferNew ? "new" : "existing"

    // Delay for animation
    DispatchQueue.main.asyncAfter(deadline: .now() + 0.4) {
        var newLow = low
        var newHigh = high

        if preferNew {
            newHigh = compareIndex  // New book is better
        } else {
            newLow = compareIndex + 1  // Existing is better
        }

        if newLow >= newHigh {
            // Done — return position within tier (0-based)
            UINotificationFeedbackGenerator().notificationOccurred(.success)
            onComplete(newLow)
        } else {
            withAnimation(.spring(response: 0.4, dampingFraction: 0.8)) {
                low = newLow
                high = newHigh
                compareIndex = (newLow + newHigh) / 2
                selectedSide = nil
            }
        }

        isAnimating = false
    }
}
```
