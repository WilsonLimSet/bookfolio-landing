Web app reference for feed queries:
- Friends tab: fetch user_books WHERE user_id IN (following IDs), ORDER BY created_at DESC LIMIT 50
  - Also fetch profiles for those users (username, avatar_url)
  - Also fetch review_likes counts and whether current user liked each review
- You tab: fetch user_books WHERE user_id = currentUser, ORDER BY created_at DESC LIMIT 50
- Incoming tab: fetch notifications WHERE user_id = currentUser, ORDER BY created_at DESC LIMIT 50
  - Mark all unread as read when tab opens
  - Fetch profiles for from_user_id

```swift
struct FeedItem: Identifiable {
    let id: UUID          // user_books.id
    let userId: UUID
    let username: String
    let avatarUrl: String?
    let title: String
    let author: String?
    let coverUrl: String?
    let openLibraryKey: String
    let score: Double
    let tier: BookTier
    let category: BookCategory
    let reviewText: String?
    let finishedAt: Date?
    let createdAt: Date
    var likeCount: Int
    var isLikedByMe: Bool
}

struct NotificationItem: Identifiable {
    let id: UUID
    let type: NotificationType
    let fromUsername: String?
    let fromAvatarUrl: String?
    let bookTitle: String?
    let bookKey: String?
    let reviewId: UUID?
    let read: Bool
    let createdAt: Date
}
```

Methods:

1. `loadFriendsActivity(userId: UUID)` async:
   - Query `follows` table: `.select("following_id").eq("follower_id", value: userId)` to get following IDs
   - If empty, set friendsActivity = [] and return
   - Query `user_books` with `.in("user_id", values: followingIds)`, ordered by created_at desc, limit 50
   - Query `profiles` for those user IDs (id, username, avatar_url)
   - Query `review_likes`: count per review_id AND check which ones current user liked
   - Map into FeedItem array joining userbook + profile + likes
   - IMPORTANT: For review_likes counts, use two queries:
     a. `.select("review_id").eq("review_id", value: ...)` grouped — OR simpler: fetch all review_likes for these review IDs and count in Swift
     b. `.select("review_id").eq("user_id", value: userId)` to check current user's likes

2. `loadYourActivity(userId: UUID)` async:
   - Query `user_books` WHERE user_id = userId, ordered by created_at desc, limit 50
   - Query `profiles` for userId to get username/avatar
   - Query review_likes for these review IDs
   - Map into FeedItem array

3. `loadNotifications(userId: UUID)` async:
   - Query `notifications` WHERE user_id = userId, ordered by created_at desc, limit 50
   - Collect unique from_user_id values
   - Query `profiles` for those IDs
   - Map into NotificationItem array
   - Mark unread as read: `.update(["read": true]).eq("user_id", value: userId).eq("read", value: false)`

4. `toggleLike(reviewId: UUID, userId: UUID, isCurrentlyLiked: Bool)` async:
   - If liked: delete from review_likes WHERE user_id AND review_id
   - If not liked: insert into review_likes
   - Update the FeedItem in friendsActivity/yourActivity arrays (likeCount ± 1, toggle isLikedByMe)
   - On unlike error, revert optimistic update

IMPORTANT: Use supabase-swift query syntax. For filtering by array of IDs, use `.in("column", values: arrayOfStrings)` where UUIDs are converted to strings.
IMPORTANT: Supabase select returns Decodable arrays. Create lightweight Decodable structs for query responses where needed (e.g., `FollowResponse` with just `followingId`).
IMPORTANT: For the review_likes join approach — fetching all likes for a set of review IDs is simpler than trying complex JOINs in supabase-swift. Fetch likes separately and merge in Swift.
