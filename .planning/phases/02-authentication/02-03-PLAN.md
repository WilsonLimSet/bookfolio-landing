3. Username validation:
   - Client-side: regex `^[a-zA-Z0-9_]{3,}$`
   - Server-side: Check profiles table for existing username
   - Debounce availability check (use Task with 0.5s delay, cancel previous)

4. Submit action:
   ```swift
   func setupUsername() async {
       isLoading = true
       defer { isLoading = false }

       do {
           // Update profile with username
           try await supabase.from("profiles")
               .update(ProfileUpdate(username: username))
               .eq("id", value: user.id)
               .execute()

           // Refresh auth state to transition to .authenticated
           await authService.checkUsernameAndSetState(user: user)
       } catch {
           errorMessage = "Failed to set username. Please try again."
       }
   }
   ```

5. Availability check with debounce:
   ```swift
   @State private var checkTask: Task<Void, Never>?

   func checkAvailability() {
       checkTask?.cancel()
       isAvailable = nil

       guard isValidUsername(username) else { return }

       checkTask = Task {
           try? await Task.sleep(for: .milliseconds(500))
           guard !Task.isCancelled else { return }

           let existing: [Profile] = try await supabase.from("profiles")
               .select("username")
               .eq("username", value: username)
               .execute()
               .value
           isAvailable = existing.isEmpty
       }
   }
   ```

IMPORTANT: ProfileUpdate must be an Encodable struct that only sends the `username` field. Check if it exists from Plan 01-02 models — if not, create a minimal one here.
IMPORTANT: The `user` parameter comes from `AuthService.AuthState.needsUsername(User)`.
